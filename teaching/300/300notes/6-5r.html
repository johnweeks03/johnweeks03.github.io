6.5.1 Motivation: Clock Arithmetic

In this section, we look at a particular family of equivalence relations on the integers and explore the way in which arithmetic interacts with them.

Problem 6.5.1.

Someone asks you, “How do you determine what time is 5 hours after 8pm?” How would you respond?

Problem 6.5.2.

Another asks, “How do you determine what time is 17 hours before 3pm?” How would you respond?

In your explanations of each of these examples, you are doing addition and subtraction modulo 12. “Modulo” arithmetic situates the numbers in a circle:

For example, “adding 5 to 8” yields 13, but “adding 5 to 8 on a clock” means we must also subtract 12 to get “5+8≡1 modulo 12”. (Modulo arithmetic doesn’t have “am” or “pm”! It also uses ≡ to distinguish from =.)

“Subtracting 17 from 3” yields −14, but “subtracting 17 from 3 on a clock” means we must add 12 to our answer until we are between 1 and 12. So we must add 12 twice and get “3−17≡10 modulo 12”.

6.5.2 Definitions

This motivates the following definitions for modular arithmetic:

Definition 6.5.3.

For each n∈ℕ, define n⁢ℤ to be the set of all integers that are divisible by n. In set-builder notation, we have

n⁢ℤ:={m∈ℤ∣m=n⁢k⁢ for some ⁢k∈ℤ}.

For example, 5⁢ℤ={…,−10,−5,0,5,10,…} and 2⁢ℤ is the set of even integers.

Example 6.5.4.

Consider the sets 3⁢ℤ, 5⁢ℤ, 15⁢ℤ, and 20⁢ℤ.

  1. a.

    List at least five elements in each of the above sets.

  2. b.

    Notice that 3⁢ℤ∩5⁢ℤ=n⁢ℤ for some n∈ℕ. What is n? Describe 15⁢ℤ∩20⁢ℤ in a similar way.

  3. c.

    Draw a Venn diagram illustrating how the sets 3⁢ℤ, 5⁢ℤ, and 15⁢ℤ intersect.

  4. d.

    Draw a Venn diagram illustrating how the sets 5⁢ℤ, 15⁢ℤ, and 20⁢ℤ intersect.

Theorem 6.5.5.

Let n∈ℕ. If a,b∈n⁢ℤ, then −a,a+b,and ⁢a⁢b are also in n⁢ℤ.

Definition 6.5.6.

A relation R from A to B is a subset of A×B.

Definition 6.5.7.

For each n∈ℕ, define the relation ≡n on ℤ via (a,b)∈≡n if a−b∈n⁢ℤ. We typically write a≡nb and say that “a is congruent to b modulo n”.

Note that a−b∈n⁢ℤ if and only if n divides a−b, which implies that a≡nb if and only if n divides a−b.

Example 6.5.8.

We consider the relation ≡5. Then we note that following:

⋯≡5−10≡5−5≡5 0≡55≡510≡5⋯
⋯≡5−9≡5−4≡5 1≡56≡511≡5⋯
⋯≡5−8≡5−3≡5 2≡57≡512≡5⋯
⋯≡5−7≡5−2≡5 3≡58≡513≡5⋯
⋯≡5−6≡5−1≡5 4≡59≡514≡5⋯

Definition 6.5.9.

We define [k]n to be the set of all integers that are congruent to k modulo n. For example, [0]n={…,−10,−5,0,5,10⁢…}. The set [5]n equals [0]n.

Example 6.5.10.

Let’s compute [2]7. Tracing back through the definitions, we see that

m∈[2]7 ⇔m≡72
⇔m−2∈7⁢ℤ
⇔m−2=7⁢k⁢ for some ⁢k∈ℤ
⇔m=7⁢k+2⁢ for some ⁢k∈ℤ.

Since the multiples of 7 are 7⁢ℤ={…,−14,−7,0,7,14,…}, we can find [2]7 by adding 2 to each element of 7⁢ℤ to get

[2]7={…,−12,−5,2,9,16,…}.

Theorem 6.5.14.

For n∈ℕ and a,b∈ℤ, [a]n=[b]n if and only if n divides a−b.

Corollary 6.5.15.

For n∈ℕ and a∈ℤ, [a]n=[0]n if and only if n divides a.

Theorem 6.5.16.

For n∈ℕ and a,b∈ℤ, [a]n=[b]n if and only if a and b have the same remainder when divided by n.

6.5.3 Addition and Subtraction Modulo n

We are now ready to define addition and multiplication modulo n. For a proof of the theorem below, use one or more of the theorems given above.

Theorem 6.5.17.

Let n∈ℕ and let a1,a2,b1,b2∈ℤ. If [a1]n=[a2]n and [b1]n=[b2]n, then

  1. a.

    [a1+b1]n=[a2+b2]n

  2. b.

    [a1⋅b1]n=[a2⋅b2]n

The previous theorem allows us to define addition and multiplication in ℤ/n⁢ℤ.

Definition 6.5.18.

Let n∈ℕ. We define the sum and product of congruence classes in ℤ/n⁢ℤ via

[a]n+[b]n:=[a+b]n⁢and⁢[a]n⋅[b]n:=[a⋅b]n.

Example 6.5.19.

By our above definition, [2]7+[6]7=[2+6]7=[8]7. But we know that [8]7=[1]7, so [2]7+[6]7=[1]7. Similarly, [2]7⋅[6]7=[12]7=[5]7.

Addition and multiplication for ℤ/n⁢ℤ has many familiar - and some not so familiar - properties. For example, addition and multiplication of congruence classes are both associative and commutative. However, it is possible for [a]n⋅[b]n=[0]n even when [a]n≠[0]n and [b]n≠[0]n. Use induction to prove the statements in the theorem below:

Theorem 6.5.20.

Let n∈ℕ. For all k∈ℕ, if [a1]n,[a2]n,…,[ak]n∈ℤ/n⁢ℤ, then

  1. a.

    [a1]n+[a2]n+⋯+[ak]n=[a1+a2+⋯+ak]n, and

  2. b.

    [a1]n⁢[a2]n⁢⋯⁢[ak]n=[a1⁢a2⁢⋯⁢ak]n.

6.4.4 Computing with Modular Arithmetic

The next result is a special case of statement (b) in the theorem above.

Corollary 6.4.1.

Let n∈ℕ. If a∈ℤ and k∈ℕ, then ([a]n)k=[ak]n.

Example 6.4.2.

Let’s compute [8179]7. We see that

[8179]7 =([8]7)179
=([1]7)179
=[1179]7
=[1]7.

For Part (a) in the next problem, use the fact that [6]7=[−1]7. For Part (b), use the fact that [23]7=[1]7.

Theorem 6.4.5.

If n∈ℕ such that n is not prime, then there exists [a]n, [b]n∈ℤ/n⁢ℤ such that [a]n⋅[b]n=[0]n while [a]n≠[0]n and [b]n≠[0]n.

We finish this section with the proof of a popular divisibility rule.

Theorem 6.4.6.

If m∈ℕ such that

m=ak⁢10k+ak−1⁢10k−1+⋯+a1⁢10+a0,

where ak,ak−1,…,a1,a0∈{0,1,…,9} (i.e., ak,ak−1,…,a1,a0 are the digits of m), then

[m]3=[ak+ak−1+⋯+a1+a0]3.

Corollary 6.4.7.

An integer is divisible by 3 if any only if the sum of its digits is divisible by 3.
